# 5. 횡단 비용·처리량 모델

> 과제는 "정밀 산정"이 아니라 **기술 스택 선택의 가이드라인**으로 비용을 본다. 여기서는 *방어 가능한 가정에 근거한 개략 추정* 으로, 각 문제가 1억 안에서 어떻게 성립하는지 보인다. 모든 단가는 2026-06 공식 소스 기준(부록 출처).

---

## 5.1 가정 (명시)

| 항목 | 가정 |
|---|---|
| 환율 | 1 USD = 1,350 KRW → **1억원 ≈ $74,000** |
| 상품 1건 토큰 | 텍스트 input ~1.5K + 이미지 ~1.0K(1장) + output ~0.3K |
| self-host | RunPod A100 SXM($1.49/hr)·L4($0.39/hr), 피크 대비 70% 효율 |
| batch | 상용 API batch 50% 할인(대량 오프라인이라 거의 항상 적용) |
| 처리 단위 | [1]·[2-2] = 상품 500만 / [2-1] = 카테고리 1만 |

> 추정 오차: throughput·이미지 토큰은 ±30~50% 변동 가능. 결론(자릿수)은 견고하다.

## 5.2 단위 비용 — self-host vs 상용 API

**self-host 500만 건 (작업별):**
| 작업 | 처리량 | 500만 비용 | 환산 |
|---|---|---|---|
| 임베딩(BGE-M3/KURE) | ~420 it/s (A100) | **~$5** | ~6,600원 |
| 경량 분류(BERT급) | ~5,600 it/s | **~$0.5** | ~700원 |
| 7B LLM 텍스트 | ~42 req/s | **~$50** | ~6.7만원 |
| VLM(Qwen2.5-VL, 이미지1장) | ~6~12 img/s | **~$350** | ~47만원 |

> 비용 지배 순서: **VLM ≫ 7B LLM ≫ 임베딩 ≫ 경량분류.** → 이미지(VLM)를 언제 쓰느냐가 비용의 핵심.

**상용 API 500만 건 (텍스트+이미지+출력, batch):**

| GPT-5.4-nano | $488 | ~$2,440 | ✅ |
| Gemini 2.5 Flash | $825 | ~$4,125 | ✅ |
| Claude Haiku 4.5 | $2,250 | ~$11,250 | ✅ |

## 5.6 ⭐ Naive vs Smart — 1억을 지키는 한 장

같은 500만 처리라도 설계에 따라 자릿수가 갈린다.

[[COST_CHART]]

| 전략 | 방식 | 500만 비용 | 1억 대비 |
|---|---|---|---|
| **Naive 1** | 상용 플래그십 VLM(Opus/GPT-5.5) 전수 | **~$56K~60K** | 단일로도 빠듯, ×3 task면 **초과** |
| **Naive 2** | GPT-4o급 비전 전수(멀티 Key·멀티이미지) | **~$75K~150K** | ❌ **초과** |
| **Smart(본 설계)** | self-host 대량 + 하드케이스만 상용 + 2-pass | **~$1K~5K** | ✅ **5~7%** |

→ **결론: 예산은 "전수에 무엇을 쓰느냐"로 결정된다.** 깔때기(funnel) + 2-pass가 자릿수를 $100K대에서 $1K대로 끌어내린다. 절감분은 *품질(사람 검수·상용 교사·롱테일)* 에 재투자한다.

---

## 5.7 예산은 실제로 어디로 가는가

세 문제 모두 **compute는 1억의 한 자릿수 %**, 나머지는 **사람(검수·golden 구축)** 이다.

```
1억 (각 문제) ≈  compute/API  ▓░░░░░░░░░  5~20%
                 사람 검수·golden ▓▓▓▓▓▓▓░░  40~70%
                 여유(품질 재투자) ▓▓▓░░░░░░  20~40%
```

이 사실이 시사하는 바: **"어떤 GPU·모델이 싸냐"는 사실 부차적이다.** 진짜 레버는 (a) 사람 노동을 어디에 집중하나(차등 게이트), (b) 라벨/검수를 어떻게 환류해 재사용하나(active learning)다. 비용 설계 = 인적 자원 배분 설계.

## 5.8 민감도 분석 (무엇이 답을 바꾸나)

| 변수 | 영향 | 대응 |
|---|---|---|
| 하드케이스 비율(라우팅 임계값) | 상용 비용 선형 증가 | 임계값 A/B, budget cap, 저가 하드케이스 모델 |
| 이미지 의존 Key 비중 | VLM 호출량 → 비용 | 2-pass로 부분집합화 |
| 사람 검수 단가·범위 | 총비용 지배 | 차등 게이트(head 전수/tail 자동) |
| 상용 모델 등급 | 교사·하드케이스 비용 | Solar Pro 2($0.1/1M) 등 한국어 가성비 우선 |
| throughput 실측 편차(±50%) | self-host 비용 | 자릿수엔 영향 적음(이미 저렴) |

> 한 줄 요약: **compute 가정이 2배 틀려도 결론(1억 충분)은 안 바뀐다.** 결론을 바꾸는 건 *사람 검수 범위*뿐이고, 그건 차등 게이트로 통제한다.
